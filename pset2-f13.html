Problem 1¶

part (c)¶

In this part, we compute a center-difference approximation of \((cu')'\) for \(c=e^{3x}\) and \(u=\sin(x)\), and compare it to the exact answer at \(x=1\) as a function of the discretization spacing \(\Delta x\). We expect second-order convergence.

In [1]: 
using PyPlot
x=1
dx = logspace(-8,-1,50)
c(x) = exp(3x)
u(x) = sin(x)
approx = (c(x+dx/2) .* (u(x+dx)-u(x))./dx - c(x-dx/2) .* (u(x)-u(x-dx))./dx) ./ dx
exact = 3*exp(3x) * cos(x) - exp(3x) * sin(x) # = c'u' + cu''
loglog(dx, abs(approx - exact), "bo")
loglog(dx, dx.^2, "k-")
legend(["|error|", L"$\Delta x^2$"], "lower right")
xlabel(L"$\Delta x$")
ylabel("|error| in derivative")
savefig("pset2-f13-p1c.pdf")

Notice two things from this plot. First, the error points decrease at the expected \(\sim \Delta x^2\) rate, since they are parallel to the \(\Delta x^2\) line on the loglog scale. Second, once \(\Delta x\) gets \(\lesssim 10^{-4}\), the errors stop decreasing, because they become limited by roundoff errors in the computer's arithmetic (floating-point arithmetic, \(\sim 15\) significant digits), and in fact the errors actually start increasing. The problem is that the roundoff errors are getting bigger and bigger for small \(\Delta x\) because our formula subtracts nearly equal quantities, an example of "catastrophic cancellation" errors. In solving PDEs, however, this is rarely a practical issue because we almost never take \(\Delta x\) to be so small.

# we will make this a function of M so that we can easily vary M to check convergence below:
function computeAeig(M)
    L = 1
    D = diff1(M)
    dx = L / (M+1)
    x = dx*0.5:dx:L # sequence of x values from 0.5*dx to <= L in steps of dx
    C = diagm(exp(3*x)) # diagonal matrix of c(x) values at the points x
    A = -D' * C * D / dx^2
    return eig(A)
end

For the last part, you are asked to check that eigenvalue differences \(|\lambda_{2M} - \lambda_{M}|\) decrease \(\sim 1/M^2\). You can do this just by looking at the numbers, and by printing out the ratios we see that doubling \(M\) decreases this difference by almost exactly a factor of 4, but for the solutions it is a bit prettier to show a plot:

x = linspace(0,L,200)

Finally, we'll use the quadgk function in Julia, which performs integrals numerically, to compute \(\int_0^L u_1(x) u_2(x) dx\). The abstol=1e-13 parameter indicates that we want to compute the integral to at least 13 decimal places if possible. The quadgk function returns a pair \((I,E)\) of values, where \(I\) is the estimated integral and \(E\) is the estimated error.

In [7]: 
quadgk(x -> conj(u1(x)) * u2(x), 0, L, abstol=1e-13)
Out[7]:
(-7.494005416219807e-16,3.319566843629218e-14)

As desired, the estimated integral \(-7.5 \times 10^{-16}\) is essentially zero up to round-off errors.
